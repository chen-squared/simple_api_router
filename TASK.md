# simple_api_router

## 任务

完成一个简单的api router，要求
- 从config读取设定的各api的信息和配置
- 接受来自openai或anthropic格式的请求
- 根据配置设定选择目标api，将请求完整地传透给目标api
- 向原请求按请求原格式发送目标api的回应
- 各api实现出错重试功能，重试多次会触发冷却。各api可设定各错误的失败允许次数，可设定触发冷却的重试次数
- 各api实现用量控制功能，可限定rpm，可限定每日/每5h/每周用量，可统计请求总次数（包含成功和失败），总用量超过且429后给定时间范围内不再尝试
- 支持分组，允许组嵌套，且组内和同一深度的组间支持按顺序请求（直到第一个不可用才去使用第二个）或负载均衡（按给定rpm平均或简单平均）
- 创建log以方便debug。log需要包含必要信息。

## 要求

- 实现后自行测试，测试时用mock api代替真实api模拟返回各种错误信息。
- 自行模拟用户使用的场景测试，测试功能完整性
- 创建并使用子代理帮助你进行测试，听取他们的反馈并继续改进，尽可能迭代多次，直到子代理和你都觉得没有改进空间了才算任务结束。
- 将总结写入PROGRESS.md文件。每次迭代都要追加写入，并记录时间，不要修改记录。
